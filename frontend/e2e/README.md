# E2E Tests

## Setup

1. Install Playwright browsers:
   ```bash
   npx playwright install
   ```

## Running Auth Setup

Before running tests with authentication, you need to generate auth files:

```bash
# Generate admin auth file
npx playwright test admin.setup.ts

# Generate user auth file
npx playwright test user.setup.ts
```

**Important:** Update credentials in setup files first:
- `e2e/admin.setup.ts` - Update `ADMIN_USERNAME` and `ADMIN_PASSWORD`
- `e2e/user.setup.ts` - Update `USER_USERNAME` and `USER_PASSWORD`

## Running Tests

### Run all tests (requires auth files)
```bash
npm run test:e2e
```

### Run specific test file
```bash
npx playwright test e2e/auth.spec.js
npx playwright test e2e/admin.spec.ts
npx playwright test e2e/user.spec.ts
```

### Run tests with UI mode
```bash
npm run test:e2e:ui
```

## Page Object Model

```
e2e/
├── pages/
│   ├── index.js
│   ├── LoginPage.js
│   └── RegisterPage.js
├── .auth/
│   ├── admin.json      # Generated by admin.setup.ts
│   └── user.json       # Generated by user.setup.ts
├── auth.spec.js        # Authentication tests (no auth required)
├── admin.spec.ts       # Admin tests (requires admin.json)
├── user.spec.ts        # User tests (requires user.json)
├── admin.setup.ts      # Generate admin.json
└── user.setup.ts      # Generate user.json
```

## Test Files

| File | Purpose | Auth Required |
|------|---------|---------------|
| `auth.spec.js` | Login/Register forms & validation | No |
| `admin.spec.ts` | Admin panel tests | Yes (admin.json) |
| `user.spec.ts` | User access control tests | Yes (user.json) |
